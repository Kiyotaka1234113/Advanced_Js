## Использованные паттерны (Ch. 9)

### 1. Promise-Based HTTP Client
Класс `PromiseHttpClient` инкапсулирует логику `fetch`. 
- **Зачем:** Централизованная обработка заголовков, автоматический парсинг JSON и типизация ошибок через `HttpError`.
- **Преимущество:** Избавляет от дублирования кода `response.ok` и `response.json()` в каждом запросе.

### 2. Retry with Exponential Backoff
Функция `withRetry` позволяет автоматически повторять запросы при временных сбоях (ошибки 429, 500, 503).
- **Алгоритм:** Задержка между попытками растет экспоненциально ($delay = baseDelay \times multiplier^{attempt}$), чтобы снизить нагрузку на сервер в моменты его нестабильности.
- **Применение:** Идеально для мобильных сетей и высоконагруженных API.

### 3. Async Queue (Concurrency Control)
Класс `AsyncQueue` ограничивает количество одновременно выполняемых промисов.
- **Зачем:** Предотвращение "забивания" сетевого канала и соблюдение лимитов (Rate Limiting) сервера.
- **Результат:** Если добавить 100 задач с лимитом 3, в каждый момент времени будут активны только 3, остальные ждут в очереди.
