## Использованные паттерны (Ch. 9)

### 1. Parallel Execution & Partial Failures
Использование `Promise.all` для параллельной загрузки данных пользователя (профиль, посты, подписчики).
- **Graceful Degradation:** Применение `Promise.allSettled` в методе `fetchMultipleUsers` позволяет приложению работать, даже если часть запросов завершилась ошибкой (мы отображаем успешных пользователей и список ID тех, кто не загрузился).



### 2. Async Data Pipeline
Класс `AsyncPipeline` реализует последовательную обработку данных через набор "стадий" (middlewares).
- **Зачем:** Позволяет разбивать сложную логику трансформации данных на мелкие, тестируемые асинхронные этапы.
- **Пример:** Загрузка -> Очистка (trim) -> Форматирование (uppercase) -> Сохранение.

### 3. Cancellation with AbortController
Реализация механизма отмены активных запросов.
- **Зачем:** Если пользователь быстро переключается между вкладками или нажимает "Отмена", `AbortController` мгновенно обрывает соединение.
- **Преимущество:** Предотвращает утечки памяти и race conditions (когда старый запрос приходит позже нового и перезаписывает актуальные данные).
